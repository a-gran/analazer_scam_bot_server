# Техническое задание для команды 1

## Проект

Telegram-бот «Кибердетектив: Распознай скам!» на `aiogram`.

## Цель задачи

Добавить управление игрой во время прохождения вопросов:

1. Кнопку «Стоп».
2. Кнопку «Перезапуск».
3. Общие улучшения интерфейса и функционала на выбор команды.

После доработки во время игры под каждым вопросом должны быть видны 4 кнопки:

- «Скам»
- «Безопасно»
- «Стоп»
- «Перезапуск»

## Задача 1. Кнопка «Стоп»

### Описание

Нужно добавить кнопку, которая останавливает текущую игру досрочно.

### Поведение

Когда пользователь нажимает «Стоп»:

1. Бот прекращает текущую игру.
2. Бот выводит текущий результат игрока.
3. Внизу итогового сообщения должна быть кнопка «Играть снова».
4. Состояние игры должно перейти в `GameStates.game_over`.

### Что считать текущим результатом

В итоговом сообщении нужно показать:

- количество правильных ответов;
- текущий счет;
- количество оставшихся жизней;
- количество уже пройденных вопросов;
- общее количество вопросов в игре;
- итоговое звание игрока;
- список признаков скама.

### Ожидаемый сценарий

1. Пользователь нажимает `/start`.
2. Пользователь нажимает «Начать игру».
3. Бот показывает вопрос и 4 кнопки.
4. Пользователь нажимает «Стоп».
5. Бот удаляет кнопки у текущего вопроса.
6. Бот показывает итоговый экран.
7. Под итоговым экраном есть кнопка «Играть снова».

## Задача 2. Кнопка «Перезапуск»

### Описание

Нужно добавить кнопку, которая сразу начинает новую игру.

### Поведение

Когда пользователь нажимает «Перезапуск»:

1. Бот прекращает текущую сессию.
2. Бот создает новую игровую сессию.
3. Очки, жизни, номер вопроса и правильные ответы сбрасываются.
4. Бот сразу показывает первый вопрос новой игры.
5. Состояние остается `GameStates.playing`.

### Ожидаемый сценарий

1. Пользователь находится в середине игры.
2. Пользователь нажимает «Перезапуск».
3. Бот удаляет кнопки у старого вопроса.
4. Бот пишет короткое сообщение, что игра началась заново.
5. Бот показывает первый вопрос новой игры.

## Файлы, которые, вероятно, нужно изменить

- `keyboards.py` — добавить кнопки «Стоп» и «Перезапуск» в игровую клавиатуру.
- `game.py` — добавить обработчики callback-кнопок.
- `session.py` — при необходимости добавить вспомогательные функции для сброса сессии.
- `results.py` — убедиться, что итоговый экран подходит для досрочного завершения.

## Где именно работать в коде

### Файл `keyboards.py`

Найдите функцию `answer_keyboard()`.

Сейчас она создает клавиатуру для ответа на вопрос. В ней уже есть кнопки:

- «Скам»
- «Безопасно»

Нужно расширить эту клавиатуру так, чтобы во время игры появились еще две кнопки:

- «Стоп»
- «Перезапуск»

Рекомендуемая структура клавиатуры:

- первая строка: «Скам» и «Безопасно»;
- вторая строка: «Стоп» и «Перезапуск».

Так пользователю будет проще отличать ответы от команд управления игрой.

### Файл `game.py`

В этом файле уже есть обработчик ответов игрока:

- `handle_answer(...)`

Рядом с ним нужно добавить два новых обработчика:

- обработчик кнопки `stop_game`;
- обработчик кнопки `restart_game`.

Обработчик «Стоп» должен:

1. Подтвердить нажатие кнопки через `await callback.answer()`.
2. Убрать кнопки у текущего вопроса через `await callback.message.edit_reply_markup()`.
3. Достать текущую сессию из `FSMContext`.
4. Перевести состояние в `GameStates.game_over`.
5. Вызвать итоговый экран из `results.py`.

Обработчик «Перезапуск» должен:

1. Подтвердить нажатие кнопки.
2. Убрать кнопки у текущего вопроса.
3. Создать новую сессию через `new_session()`.
4. Сохранить новую сессию в `FSMContext`.
5. Установить состояние `GameStates.playing`.
6. Отправить сообщение «Игра перезапущена».
7. Показать первый вопрос новой игры через `send_question(...)`.

### Файл `results.py`

Проверьте функцию `show_results(...)`.

Она уже показывает итог игры. Ее можно использовать и для обычного конца игры, и для кнопки «Стоп».

Если в итогах нет информации о том, сколько вопросов игрок успел пройти, добавьте эту строку.

Пример смысла строки:

`Пройдено вопросов: 5 из 20`

### Файл `session.py`

Скорее всего, менять этот файл не обязательно.

Но если команде удобно, можно добавить маленькую функцию-обертку для перезапуска игры. Например, функция может просто вызывать `new_session()`.

Это необязательно, но может сделать код понятнее.

## Пошаговый план выполнения

1. Откройте `keyboards.py`.
2. Добавьте две новые кнопки в `answer_keyboard()`.
3. Проверьте, что у кнопок есть разные `callback_data`.
4. Откройте `game.py`.
5. Добавьте обработчик для `stop_game`.
6. Добавьте обработчик для `restart_game`.
7. Проверьте, что оба обработчика работают только в состоянии `GameStates.playing`.
8. Запустите бота.
9. Пройдите ручное тестирование по чек-листу ниже.

## Callback data

Рекомендуемые значения:

- `stop_game` — для кнопки «Стоп».
- `restart_game` — для кнопки «Перезапуск».

Важно: значения `callback_data` в `keyboards.py` должны точно совпадать со значениями в обработчиках `game.py`.

Если в кнопке написано `callback_data="stop_game"`, то обработчик тоже должен проверять именно `"stop_game"`.

## Как тестировать

Перед запуском бота выполните:

```bash
.venv/bin/python -m py_compile bot.py config.py start.py game.py answer.py session.py results.py states.py keyboards.py data.py
```

### Проверка запуска

Запустите бота:

```bash
.venv/bin/python bot.py
```

В консоли должно появиться сообщение, что бот запущен.

## Распространенные ошибки

### Ошибка 1. Кнопка есть, но ничего не происходит

Скорее всего, `callback_data` в `keyboards.py` не совпадает с проверкой в `game.py`.

Например, в кнопке написано `restart`, а обработчик ждет `restart_game`.

### Ошибка 2. Бот падает с ошибкой `KeyError: 'session'`

Так бывает, если обработчик пытается достать сессию, когда игра еще не началась.

Проверьте, что обработчики «Стоп» и «Перезапуск» работают только в состоянии `GameStates.playing`.

### Ошибка 3. После нажатия кнопки ее можно нажать еще раз

Значит, забыли убрать старую клавиатуру.

В обработчике должна быть строка:

```python
await callback.message.edit_reply_markup()
```

### Ошибка 4. После «Стоп» игра продолжает принимать ответы

Значит, состояние не перевели в `GameStates.game_over`.

После остановки игры нужно вызвать:

```python
await state.set_state(GameStates.game_over)
```

### Ошибка 5. Перезапуск не сбрасывает счет

Значит, команда использует старую сессию вместо новой.

Для перезапуска нужно создать новую сессию через `new_session()` и сохранить ее в `FSMContext`.

## Общее задание для команды

Команда должна внедрить любое улучшение интерфейса или функционала на свой вкус.

Примеры:

- сделать текст вопросов визуально понятнее;
- добавить короткое сообщение при перезапуске;
- улучшить итоговый экран;
- добавить эмодзи или более аккуратное форматирование;
- защитить кнопки от повторного нажатия.

## Критерии готовности

- Во время игры отображаются 4 кнопки.
- Кнопка «Стоп» завершает игру и показывает текущий результат.
- После «Стоп» доступна кнопка «Играть снова».
- Кнопка «Перезапуск» начинает игру заново без команды `/start`.
- Старые кнопки удаляются после нажатия.
- Нет ошибок в консоли при обычном игровом сценарии.
- Код проходит проверку импорта и запуска.
